## 기본 계획
🛠️ Phase 1. 데이터 전처리 (Data Engineering)
AI가 소화할 수 있도록 두 데이터셋을 정제하는 가장 중요한 뼈대 작업입니다.

1. train_2.csv (시계열 데이터 가공)

시간 변수 추출: 일시 컬럼(예: 20240601 00)을 쪼개서 월(Month), 일(Day), 시간(Hour), 요일(Day of Week), 주말 여부 컬럼을 파생 변수로 만듭니다.

기상 파생 변수 생성: 기온(°C)과 습도(%) 데이터를 수식에 넣어 '불쾌지수(THI)' 컬럼을 자체적으로 추가합니다. (여름철 피크 예측의 핵심 치트키입니다.)

2. building_info_3.csv (스펙 데이터 가공)

원-핫 인코딩: 건물유형에 있는 '호텔', '상용', '병원' 등의 글자를 AI가 이해할 수 있도록 0과 1의 컬럼들로 변환합니다. (예: is_호텔: 1, is_병원: 0)

결측치 처리: 태양광용량, ESS저장용량 등에 적힌 하이픈(-) 기호는 0으로 치환하거나, 데이터가 너무 없다면 해당 컬럼을 과감히 Drop 합니다.

🧩 Phase 2. 정답지 만들기 (K-Means 시계열 군집화)
건물의 '면적'이 아니라 순수하게 '전기를 쓰는 모양(패턴)'을 기준으로 건물을 묶어냅니다.

입력 데이터: train_2.csv

학습 로직:

건물별(1~100번)로 시간대(0시~23시)에 따른 전력 소비량의 평균치를 구합니다.

[주의!] 정규화(MinMax Scaling): 연면적이 큰 병원(5,000kWh)과 작은 학교(100kWh)를 그냥 넣으면 '크기'로 묶여버립니다. 모든 건물의 최대 전력량을 1로, 최소를 0으로 압축(Scaling)하여 오직 '곡선의 형태'만 남깁니다.

군집화 적용: 정규화된 100개의 패턴을 K-Means 알고리즘에 넣습니다. 실루엣 점수를 확인하며 최적의 방 개수(예: K=4)를 확정합니다.

산출물 (Output):

"1번 건물 = 그룹 0, 2번 건물 = 그룹 2" 형태의 매핑 테이블 (이것이 분류 모델의 정답지가 됩니다.)

🚦 Phase 3. 네비게이션 만들기 (Random Forest 분류 모델)
MVP의 핵심인 '신규 사용자 콜드 스타트 방어 로직'을 학습하는 단계입니다.

입력 데이터: 가공된 building_info_3.csv (X) + Phase 2에서 만든 매핑 테이블 (Y)

학습 로직:

AI(Random Forest 또는 LightGBM Classifier)에게 문제(면적, 냉방 면적, 건물유형)와 정답(이 건물이 속한 군집 번호)을 던져주고 규칙을 찾게 합니다.

"호텔이고 연면적이 8만 제곱미터 이상이면 높은 확률로 그룹 2(심야 피크형)구나!" 라는 것을 학습합니다.

산출물 (Output):

Classifier_Router_Model.pkl (추후 백엔드 FastAPI에서 신규 유저 가입 즉시 실행될 네비게이터)

🎯 Phase 4. 최종 예측기 만들기 (LightGBM 회귀 모델)
가장 비싼 시간대의 요금 폭탄을 막아줄 실제 전력량 예측 엔진을 구축합니다.

입력 데이터: 군집별로 분할된 train_2.csv

학습 로직:

전체 100개 건물 데이터를 Phase 2에서 나눈 군집(예: 4개) 기준으로 완전히 쪼갭니다.

모델을 1개가 아니라 4개의 LightGBM Regressor로 각각 따로 학습시킵니다.

X(특성): 시간, 요일, 기온, 습도, 풍속, 일사량, 불쾌지수

Y(정답): 실제 전력소비량(kWh)

평가 지표로는 대형 오차를 막기 위해 RMSE를 손실 함수로 최적화하고, 검증 시에는 SMAPE(0에 가까울 때의 왜곡 방지)를 사용합니다.

산출물 (Output):

LGBM_Regressor_Cluster_0.pkl

LGBM_Regressor_Cluster_1.pkl 등 총 4개의 예측 모델 파일.
